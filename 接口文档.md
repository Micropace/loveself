### 后台API说明文档

### 答题相关API
#### 1. 首页内容获取API
- 用户首次扫码，后台将根据openid创建用户记录
```
GET: /api/homepage
URL参数：
{
    scene: '',  // 扫码得到的场景值
    openid: '', // 用户openid
}

成功返回：
{
    status: 'success',
    data: {
        isFirstScan: 0,              // 用户是否首次扫码， 0：否，1：是
        bannerText: 'xxxxx',         // banner标题，不存在图片则选用这个
        bannerPicName: 'xxx.jpg',    // banner图片
        homePagePicName: 'xxxx.jpg', // 显示时拼装为: host/xxx.jpg
        homePageDiscription: 'xxxxx',// 首页描述文字，有则显示，无则只显示图片
    }
}
失败返回：
{
    status: 'error',
    message: 'xxx',
}
```

#### 2. 手机号提交API
- 提交手机号后，后台将根据手机号创建代表记录。
- 如代表创建成功或者已存在，将创建用户和代表的关联关系。
- 重复提交，后台将检查两者是否关联，是则忽略，否则关联。
```
POST: /api/register
POST参数：
{
    openid: '',  // 用户openid
    mobile: '',  // 填写的手机号
}
成功返回：
{
    status: 'success',
}
失败返回：
{
    status: 'error',
    message: 'xxx',
}
```
#### 3. 题库内容获取API
- 题干、选项：可能是文字，可能是图片，或者两者都存在。
- 所以下面题目列表中`content`字段和`picName`字段是动态的。
前端需要判断一下字段是否存在，存在字段则显示相应内容。

```
GET: /api/questions
URL参数：
{
    scene: '',  // 扫码得到的场景值
    openid: '', // 用户openid
}
成功返回：
{
    status: 'success',
    data: {
        bannerText: 'xxxxx',     // banner标题，不存在图片则选用这个
        bannerPicName: 'xx.jpg', // banner图片
        questions: [
            {
                index: 1,        // 题目序号
                type: '',        // 题目类型，单选single_select|多选multi_select|开放open
                title: {
                    content: '', // 题干内容
                    picName: '', // 题干中的图片
                },
                options: [
                    {
                        index: 'A',  // 选项序号
                        content: '', // 选项内容，有文字存在则这个字段存在，无文字只显示图片，则没有这个字段。
                        picName: '', // 选项中的图片，有图片存在则这个字段存在，无图片只显示文字，则没有这个字段。
                    },
                    ...
                ],
                answers: ['A', 'B'], // 正确答案数组，值为选项的序号
            },
            ...
        ]
    }
}
失败返回：
{
    status: 'error',
    message: 'xxx',
}
```
#### 4. 答题结果提交API
- 答题结果参数`results`需前端直接传递JSON序列化后的字符串, 结果列表格式为：
```$xslt
    [
        {
            index: 1,       // 题目序号
            answers: ['A'], // 用户选择的答案
        },
        ...
    ],
```
```
POST: /api/answers
POST参数：
{
    scene: '',  // 扫码得到的场景值
    openid: '', // 用户openid
    results: '',// 结果
}
成功返回：
{
    status: 'success',
}
失败返回：
{
    status: 'error',
    message: 'xxx',
}
```

### 二维码管理相关API
#### 1. 根据场景值查询小程序码情况
```$xslt
GET: /api/qrcode
URL参数：
{
    scene: '', // 场景值
}
成功返回：
{
    status: 'success',
    data: {
        id: 12344556,   // ID索引
        scene: '',      // 场景值
        type: '',       // 类型： 永久|临时(目前只有永久的)
        name: '',       // 二维码下载到服务器后的文件名称
        path: '',       // 二维码存储在服务器的全路径
        expire: 1234,   // 临时二维码的有效时间(永久二维码该值为空)
        createdAt: '',  // 创建时间
        updatedAt: '',  // 更新时间
    }
}
失败返回：
{
    status: 'error',
    message: 'xxx',
}
```
